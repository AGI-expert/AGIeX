# AGI Expert — Node Software

The runtime that powers every node in the AGI Expert network. Pure JavaScript ESM — no TypeScript, no build step. One entry point (`main.js`) boots 9 subsystems in sequence, wires them together, and runs autonomously from there.

## What It Does

When you run `node src/main.js`, the node:

1. **Generates a cryptographic identity** — Ed25519 keypair stored in `data/identity.json`, used for peer discovery, message signing, and experiment attribution.
2. **Joins the P2P network** — Connects to peers via libp2p (TCP + WebSocket), discovers local nodes with mDNS, syncs with the wider network through bootstrap peers and Kademlia DHT.
3. **Detects hardware capabilities** — Reads `hw-profile.json` (generated by `setup/detect-hardware.sh`) and enables up to 9 services: inference, research, proxy, storage, embedding, memory, orchestration, validation, and relay.
4. **Starts an inference server** — Express API on port 8080, OpenAI-compatible (`/v1/chat/completions`, `/v1/models`, `/v1/embeddings`). Loads GGUF models via node-llama-cpp if a GPU is available.
5. **Runs proof-of-compute pulses** — Every ~90 seconds: deterministic matrix multiplication → Merkle tree → random row challenges → proof verification → AGIEX token minting on Solana.
6. **Conducts autonomous research** — Cycles through 9 research domains, mutates hyperparameters, runs experiments, tracks personal bests, and shares breakthroughs with peers.
7. **Syncs leaderboards** — Conflict-free replicated data types (Yjs CRDTs) keep every node's leaderboard consistent without a central server.
8. **Enforces network security** — Message authentication, rate limiting, challenge-response liveness checks, anomaly detection, peer scoring, and Sybil resistance.
9. **Manages research governance** — 7-seat council reviews proposals, community votes with `sqrt(reputation)` weighting, proposals flow through DRAFT → REVIEW → VOTING → ACTIVE → SUNSET.

## Development with Dev Container

The repo root contains a Dev Container for Node.js / JavaScript development (Ubuntu 22.04, Node.js 22, npm). A **separate** Dev Container lives in `program/.devcontainer/` for Rust / Solana / Anchor work — see `program/README.md`.

### Setup

1. Install [Docker](https://docs.docker.com/get-docker/) and the [Dev Containers](https://marketplace.visualstudio.com/items?itemName=ms-vscode-remote.remote-containers) VS Code extension.

2. Open the repo root in VS Code:
   ```bash
   code .
   ```

3. VS Code will detect two Dev Containers (root for Node.js, `program/` for Rust/Solana). Pick **"AGI Expert — Node.js"** for JavaScript work. Or run the command palette (`Ctrl+Shift+P`) → **Dev Containers: Reopen in Container**.

4. Wait for the container to build (first time takes a few minutes). The `postCreateCommand` installs dependencies and prints versions:
   ```
   v22.x.x
   10.x.x
   vitest/4.x.x
   ```

You're ready to develop. All paths — `src/`, `tests/`, `projects/`, `package.json` — are available at `/workspace`. The container includes Prettier, ESLint, Error Lens, and the Vitest Explorer extension.

> **Solana / Anchor work?** Close this container, then reopen in the `program/` Dev Container instead (Rust, Solana CLI, Anchor). See `program/README.md` for details.

## Boot Sequence

Each subsystem depends on the ones before it. The order matters:

| Step | Subsystem | What Happens |
|------|-----------|-------------|
| 1 | Identity | Load or generate Ed25519 keypair from `data/identity.json` |
| 2 | P2P Network | Start libp2p node (TCP + WS + mDNS + DHT + GossipSub) |
| 2b | P2P Security | Attach message auth, rate limiting, challenge-response |
| 3 | CRDT Leaderboards | Create 7 Yjs documents (one per research domain) |
| 4 | Capabilities | Auto-detect and start hardware-gated services |
| 4b | Governance | Load council state and proposals from disk |
| 5 | Inference Server | Start Express on port 8080, load GGUF model if available |
| 6 | Pulse Verification | Begin ~90s proof-of-compute loop, resume from last on-chain round |
| 7 | Agent Brain | Start 10s decision loop, initialize 9 research pipelines |
| 8 | Status Endpoints | Register `/status`, `/leaderboard/:domain`, `/snapshot`, `/security`, `/governance`, `/dashboard` on Express |

After boot, the node prints a status banner and runs autonomously until `Ctrl+C` triggers graceful shutdown (brain → auto-stake timer → security → pulse → capabilities → inference → P2P).

## Subsystem Details

### Agent Brain (`brain/agent.js`)

The central decision loop. Ticks every 10 seconds. Maintains a `Map<string, ResearchPipeline>` for 9 research projects. Each tick:

1. Checks if the node has research capability
2. Round-robins through projects
3. Runs one experiment cycle (hypothesis → train → evaluate)
4. Submits results to local CRDT leaderboard
5. Persists to DHT storage if available
6. Broadcasts results to peers via GossipSub

Also handles incoming gossip: routes experiment results as inspiration to the correct pipeline, updates CRDT leaderboards, and applies state updates from peers.

### Research Pipeline (`research/pipeline.js`)

One instance per project. Implements a 5-stage loop:

1. **Hypothesis** — Mutate the best known config. 30+ mutation strategies (learning rate, architecture depth, LoRA rank, reasoning scaffolds, etc.). 30% chance of adopting a setting from a peer's result via the inspiration buffer.
2. **Experiment** — Run the experiment (simulated in dev; calls Python training scripts in production).
3. **Paper** — Every 10 runs, generate a structured report of findings.
4. **Critique** — Score peer papers (5-10 range).
5. **Discovery** — High-scoring papers feed back as inspiration.

State persists to `projects/<name>/agents/<peerId>/best.json` and individual run files.

### P2P Networking (`p2p/node.js`, `p2p/security.js`)

**node.js** creates a libp2p node with: TCP + WebSocket transports, Noise encryption, Yamux multiplexing, GossipSub pub/sub, mDNS local discovery, Kademlia DHT, and circuit relay for NAT traversal. Exports 10 GossipSub topics under the `agi/` namespace.

**security.js** adds 7 security layers:
1. Message authentication (Ed25519 signatures on every GossipSub message)
2. Challenge-response (random peer challenges every 5 min, 10s timeout)
3. Peer attestation (3 validators must confirm pulse proofs before on-chain submission)
4. Anomaly detection (flags proofs <50ms or >30s, bot-like timing variance <5%)
5. Peer scoring (cumulative score, auto-blocklist at -100)
6. Rate limiting (60 msg/min/peer, 50 proofs/hour/peer)
7. Sybil resistance (flags 3+ peers with identical Merkle roots in the same round)

### Pulse Verification (`pulse/verification.js`)

The proof-of-compute protocol, every ~90 seconds:

1. **VRF leader election** — Deterministic from `SHA-256(networkSeed:roundNumber)` + sorted peer list
2. **Seed broadcast** — Leader distributes the round seed
3. **Matrix computation** — Each node generates two deterministic 256×256 matrices from the seed and multiplies them
4. **Merkle commitment** — SHA-256 hash of each result row → Merkle tree → root commitment
5. **Challenge** — Leader picks 4 random rows to verify
6. **Proof reveal** — Nodes provide the challenged rows + Merkle proofs
7. **Verification + reward** — Valid proofs trigger AGIEX minting on Solana

### CRDT Leaderboards (`crdt/leaderboard.js`)

7 independent Yjs documents: `research`, `search`, `finance`, `coding`, `skills`, `causes`, `agi`. Each contains a `Y.Map("entries")` keyed by peerId. State syncs over GossipSub every ~50 seconds via full-state updates. Supports `submit()`, `getTop()`, `getGlobalBest()`, `applyUpdate()`, and `snapshot()`.

### Capabilities (`capabilities/index.js`)

9 services, each extending `BaseCapability` with `start()`/`stop()`:

| Capability | Hardware Requirement | What It Does |
|------------|---------------------|-------------|
| `inference` | GPU >= 4GB VRAM | Registers node as inference provider |
| `research` | GPU >= 6GB or CPU 4c + 8GB RAM | Enables research pipeline |
| `proxy` | Public IP | HTTP proxy for peer web requests (port 4010) |
| `storage` | 50GB disk | DHT block store in `data/blocks/` |
| `embedding` | CPU 2c + 4GB RAM | 384-dim vector embeddings (MiniLM fallback) |
| `memory` | 8GB RAM + 20GB disk | In-memory cosine similarity vector store |
| `orchestration` | CPU 4c + 8GB RAM | Task decomposition and routing to capable peers |
| `validation` | CPU 2c | Pulse proof verification for peer attestation |
| `relay` | Public IP | libp2p circuit relay for NAT-blocked nodes |

Each enabled capability adds a weight bonus to reward calculations (`CAPABILITY_WEIGHTS` constant, 0.03–0.12 per capability).

### Inference Server (`inference/server.js`)

Express on port 8080 (configurable via `INFERENCE_PORT` env var). OpenAI-compatible API:

- `GET /v1/models` — List loaded models (+ any `.gguf` files in `models/`)
- `POST /v1/chat/completions` — Chat (streaming SSE + non-streaming JSON)
- `POST /v1/embeddings` — Text embeddings
- `GET /health` — Health check with model status and uptime

Loads GGUF models via `node-llama-cpp`. Falls back to proxy mode if no model is available.

### Governance (`governance/council.js`)

7-seat council with 90-day terms. Proposal lifecycle:

```
DRAFT → REVIEW (3 days, 4/7 council approval) → VOTING (7 days, 66% threshold, 10% quorum) → APPROVED → ACTIVE → SUNSET
```

Voting power: `sqrt(reputation)` to reduce plutocratic effects. Minimum 2000 reputation to propose, 500 to vote. Council seats cover: AI/ML Research, Systems Engineering, Ethics, Legal, Domain Science, Community Advocacy, Security Audit.

Proposals persist as JSON in `governance/proposals/`. Council state in `governance/state.json`. All actions propagate over GossipSub (`agi/governance` topic).

## Configuration

| Source | Purpose |
|--------|---------|
| `node-config.json` | Network config: bootstrap nodes, listen addresses, model selection |
| `hw-profile.json` | Hardware profile from `setup/detect-hardware.sh`: GPU, CPU, RAM, disk |
| `AGI_DATA_DIR` env | Data directory (default: `./data`) |
| `INFERENCE_PORT` env | API port (default: 8080) |
| `NODE_SOLANA_KEYPAIR` env | Path to Solana keypair JSON for on-chain rewards |
| `AGI_MINT_ADDRESS` env | AGIEX SPL token mint address |
| `--auto-stake` flag | Auto-stake tokens when balance >= 100 AGIEX |

## Data Flows

```
Research:  pipeline → disk (best.json) → local CRDT → GossipSub → peer CRDTs
                                                                  → inspiration buffer
                                                                  → 30% adoption in next hypothesis

Rewards:   pulse proof → P2P attestation (3 validators) → Solana program → mint AGIEX

Leaderboard:  local submit → Yjs doc → GossipSub state sync → peer Yjs docs → convergence

Security:  incoming msg → rate limit → blocklist check → route security msgs → allowed/blocked
```

## Running

```bash
# Start with defaults
node src/main.js

# Use a config file
node src/main.js --config node-config.json

# Custom data directory
AGI_DATA_DIR=./data node src/main.js

# Enable auto-staking
node src/main.js --auto-stake
```

Or use the project launcher which handles hardware detection first:

```bash
./start.sh              # standard launch
./start.sh --auto-stake # with auto-staking
```

## Dependencies

- **libp2p** ecosystem — TCP, WebSocket, Noise, Yamux, GossipSub, mDNS, DHT, circuit relay, dcutr
- **Yjs** — CRDT library for conflict-free leaderboard replication
- **Express** — HTTP server for the inference API
- **node-llama-cpp** — GGUF model loading and inference (optional, for GPU nodes)
- **@solana/web3.js** + **@solana/spl-token** — On-chain reward submission and staking
- **Node.js crypto** — Ed25519 keys, SHA-256, matrix seeding
