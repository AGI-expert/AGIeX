1 Abstract

We present AGI Expert, a fully decentralized peer-to-peer network where autonomous nodes conduct machine learning research, serve local inference, prove computational work, and earn AGIEX tokens through a trustless on-chain reward mechanism. The network requires no central coordinator, no cloud infrastructure, and no human approval for token minting. Each node auto-detects its hardware capabilities, loads quantized language models locally, participates in a 90-second pulse verification cycle, and contributes to eight parallel research projects through an evolutionary experiment pipeline. The Solana program governing AGIEX is deployed immutably with a Program-Derived Address (PDA) as sole mint authority — no private key exists that can mint tokens outside the verified proof pathway. Research direction is governed by a seven-seat multidisciplinary council and community-weighted voting, ensuring no single interest controls the network's trajectory.

2 Introduction

Contemporary AI research is concentrated in a small number of well-funded laboratories with exclusive access to compute, data, and talent. This concentration creates systemic risks: single points of failure, misaligned incentives, and barriers to participation for the global research community. Open-source model releases partially address the access problem but do not solve the coordination, incentive, or governance challenges.

AGI Expert proposes a different model: a flat, permissionless network of autonomous agents that collectively advance AI research. Every participant — from a laptop running a 1B parameter model to a server with an H100 — contributes compute, earns tokens proportional to their proven work, and has a voice in research governance. The system is designed around five core principles:

  1. No central authority. Every node is architecturally identical. There are no coordinators, no privileged roles, no admin keys.
  2. Trustless economics. The on-chain program is immutable. The mint authority is a PDA. Rewards are computed deterministically from verifiable proofs.
  3. Hardware-adaptive participation. Capabilities are auto-detected and enabled at startup. Zero configuration required.
  4. Autonomous research. Nodes generate hypotheses, run experiments, write papers, critique peers, and evolve strategies without human intervention.
  5. Diversified governance. A seven-seat council spanning AI/ML, systems engineering, ethics, law, domain science, community advocacy, and security auditing governs research direction.

This whitepaper details the architecture, cryptoeconomics, and governance of the AGI Expert network and the AGIEX token.

4 Pulse Verification Protocol

The pulse protocol is the heartbeat of the network. Every ~90 seconds, nodes prove they are performing genuine computation and submit verifiable proofs to the Solana program.

4.1 Matrix Computation Challenge

Each pulse round requires the node to compute a matrix multiplication challenge. Let s denote the round seed (a 32-byte value derived from the previous round's hash):

Given: s = SHA-256(previous_round_hash || round_number) Generate: A, B in R^{256x256} where A[i][j] = PRNG(s, i, j) Compute: C = A * B C[i][j] = sum_{k=0}^{255} A[i][k] * B[k][j] for i, j in [0, 255]

The matrices A and B are generated deterministically from the seed using SHA-256 as a PRNG, ensuring any validator can independently reproduce the computation. The matrix dimension is fixed at MATRIX_SIZE = 256.

4.2 Merkle Commitment Scheme

The result matrix C is committed via a binary Merkle tree:

Let row_i = C[i][0..255] for i in [0, 255] Leaf_i = SHA-256(row_i) // 256 leaves Node_k = SHA-256(Node_{2k} || Node_{2k+1}) // 8 levels Root = Node_0 // 32-byte commitment

The node broadcasts its Merkle root as a commitment. During verification, 4 rows are challenged:

challenge_indices = { SHA-256(s || j) mod 256 : j in [0, 3] } For each challenged row i: - Node reveals: row_i and merkle_proof_i (8 sibling hashes) - Verifier: recomputes row_i from A, B verifies SHA-256(row_i) against merkle_proof_i verifies proof_i resolves to committed root

The probability of cheating on all 4 challenges simultaneously is (1/256)^4 = 1/4,294,967,296, providing strong statistical guarantees without requiring full matrix verification.

4.3 VRF Leader Election

A Verifiable Random Function determines the round leader from the set of online peers:

peers = sort(connected_peer_ids) // deterministic ordering index = uint32(SHA-256(round_seed)) mod |peers| leader = peers[index]

The leader coordinates the commit-reveal cycle for the round. All peers can independently verify the leader selection was correct given the same seed and peer set.

4.4 Cross-Validation

Before on-chain submission, proofs must be attested by independent validators:

ParameterValue
Validators required per proof3

Validators independently compute the matmul for the challenged rows and compare against the submitter's values. This cross-validation layer prevents collusion between a dishonest node and a complicit leader.

5.2 Reward Formula

The reward for a single pulse round is computed as:

R = R_base * B_uptime * B_capability * M_reputation

Where each component is defined as follows:

5.2.1 Base Reward

R_base = 10 AGIEX

5.2.2 Uptime Bonus

The uptime bonus rewards long-running nodes with a logarithmic curve that provides diminishing returns:

B_uptime = 1 + 0.2 * ln(1 + h / 12) where h = continuous uptime in hours
UptimehB_uptime
1 hour11.016
12 hours121.139
24 hours241.220
7 days1681.546
30 days7201.826

The logarithmic shape ensures early uptime is rewarded substantially while preventing runaway bonuses for always-on infrastructure.

5.2.3 Capability Bonus

Nodes are rewarded for the services they provide to the network:

B_capability = 1 + sum(w_i) for each enabled capability i where w_i in {0.10, 0.12, 0.08, 0.06, 0.05, 0.05, 0.05, 0.04, 0.03}

With all 9 capabilities enabled: B_capability = 1.58. A minimal node with only validation enabled: B_capability = 1.04.

5.2.4 Reputation Multiplier

M_reputation = max(0.1, S_composite / 10000) S_composite = 0.30 * S_honesty + 0.30 * S_quality + 0.20 * S_loyalty + 0.20 * S_consistency Each sub-score is on a [0, 10000] scale.

The floor of 0.1 ensures that even low-reputation nodes receive 10% of the base reward, preventing complete exclusion while still strongly incentivizing good behavior.

5.2.5 Complete Reward Calculation

Combining all components, the reward per round for a node with reputation score S, uptime h hours, and enabled capability weights summing to W:

R = 10 * (1 + 0.2 * ln(1 + h/12)) * (1 + W) * max(0.1, S/10000) Bounds: Minimum (new node, 1 cap, 0.1x rep): R_min = 10 * 1.0 * 1.04 * 0.1 = 1.04 AGIEX/round Maximum (30d uptime, 9 caps, perfect rep): R_max = 10 * 1.826 * 1.58 * 1.0 = 28.85 AGIEX/round

5.3 Reputation System

Each node maintains four on-chain reputation sub-scores, all on a [0, 10000] scale:

5.3.1 Honesty Score (weight: 30%)

S_honesty = (valid_proofs / (valid_proofs + invalid_proofs)) * 10000 Initial value: 1000 Per valid proof: +5 Per invalid proof: -200

5.3.2 Quality Score (weight: 30%)

S_quality = base_quality - (consecutive_failures * 500) base_quality = (recent_valid / (recent_valid + recent_invalid)) * 10000 QUALITY_WINDOW = 100 (last 100 proofs)

5.3.3 Loyalty Score (weight: 20%)

S_loyalty = min(10000, (days_active / 30) * 10000) Linear ramp: 0 at day 0, 10000 at day 30. Threshold: 2,592,000 seconds = 30 days.

5.3.4 Consistency Score (weight: 20%)

S_consistency = max(0, 10000 - consecutive_misses * 200) MAX_CONSECUTIVE_MISSES = 10 triggers additional reputation penalty. Per missed round: -10 reputation.

5.3.5 Composite Score

S = 0.30 * S_honesty + 0.30 * S_quality + 0.20 * S_loyalty + 0.20 * S_consistency Operating thresholds: S >= 500 required to earn rewards S >= 2000 required to validate peers S >= 2000 required to submit governance proposals S >= 500 required to vote on proposals

5.4 Emission Curve & Supply Projections

AGIEX has no halving schedule, no epoch-based emission changes, and no inflationary mechanics. The emission rate is purely a function of network participation: more nodes producing valid proofs means faster emission.

5.4.1 Emission Rate Formula

E_daily = N * rounds_per_day * R_avg where: N = number of active nodes rounds_per_day = 86400 / 90 = 960 R_avg = average reward per round across all nodes

For a network of N nodes with average reward R_avg:

E_daily(N) = N * 960 * R_avg

5.4.2 Time to Supply Cap

The total supply is hardcoded at S_max = 1,000,000,000 AGIEX. The time to reach the cap at constant network size:

T_cap = S_max / E_daily(N) Assuming R_avg = 15 AGIEX (moderate rep, moderate caps): T_cap(10) = 1,000,000,000 / (10 * 960 * 15) = 6,944 days = ~19.0 years T_cap(100) = 1,000,000,000 / (100 * 960 * 15) = 694 days = ~1.9 years T_cap(1000) = 1,000,000,000 / (1000 * 960 * 15) = 69 days = ~2.3 months T_cap(10000) = 1,000,000,000 / (10000 * 960 * 15) = 6.9 days = ~1 week

5.4.3 Supply Function

Assuming a logistic network growth model N(t) = N_max / (1 + e^(-k(t - t_0))), the cumulative supply at time t:

S(t) = min(S_max, integral_0^t E_daily(N(tau)) d(tau)) = min(S_max, 960 * R_avg * integral_0^t N(tau) d(tau)) = min(S_max, 960 * R_avg * N_max * [t + (1/k) * ln((1 + e^(-k*t_0)) / (1 + e^(-k*(t - t_0))))])

This produces the characteristic S-curve: slow emission during early adoption, rapid emission during network growth, and hard termination at the 1B cap.

7.4 Sybil Resistance

The system employs layered sybil resistance:

  1. Economic barrier: 100 AGIEX minimum stake per node creates a linear cost for sybil attacks.
  2. Temporal barrier: 14-day maturation + 100 valid proofs before earning. A sybil operator must maintain each fake node for two weeks.
  3. Computational barrier: Each node must independently compute 256x256 matmul every 90 seconds. The cost of running N sybil nodes scales linearly with N.
  4. Detection: 3+ nodes with identical Merkle roots in the same round are flagged. Penalty: -100 peer score per detected sybil node.
  5. Cross-validation: Validators independently verify proofs. Collusion between sybil nodes is detected when validators disagree.
Cost of sybil attack with K nodes: C_sybil(K) = K * stake_min + K * compute_cost * T_maturation = K * 100 AGIEX + K * (electricity + hardware) * 14 days Reward for K sybil nodes (assuming detection at rate d): R_sybil(K) = K * R_avg * (1 - d) - K * slash_rate * stake where d increases rapidly with K (detection is O(K^2) in pair checks)

For any non-trivial number of sybil nodes, the expected cost exceeds the expected reward, making sybil attacks economically irrational.

Research direction is governed by a seven-seat multidisciplinary council. Each seat represents a distinct domain of expertise, ensuring no single perspective dominates decision-making:

#SeatEvaluates
1AI/ML ResearcherTechnical feasibility, architecture validity, benchmark rigor
2Systems EngineerInfrastructure requirements, scalability, resource costs
3Ethicist / PhilosopherAlignment, safety, societal impact, dual-use potential
4Legal CounselRegulatory compliance, data licensing, intellectual property
5Domain ScientistScientific rigor, methodology, reproducibility
6Community AdvocateUser impact, accessibility, practical value
7Security AuditorAttack surfaces, game theory, adversarial scenarios

Council members serve 90-day terms (7,776,000 seconds). Council approval requires 4 of 7 members.

10.2 Proposal Lifecycle

DRAFT --> REVIEW (3 days) --> VOTING (7 days) --> APPROVED / REJECTED --> ACTIVE --> SUNSET

Proposal requirements: Proposer must have reputation >= 2,000. Proposals must include: objectives, benchmarks, data requirements, ethics statement, and success criteria.

10.3 Voting Mechanics

Vote weight = sqrt(reputation) Rationale: Square root scaling reduces plutocratic influence. Node with rep 10000: weight = 100 Node with rep 2500: weight = 50 Node with rep 500: weight ~ 22 Approval threshold: 66% of weighted votes Quorum: 10% of active nodes must participate Voting period: 7 days (604,800,000 ms) Min rep to vote: 500 / 10000

This design ensures that high-reputation nodes have influence proportional to the square root of their reputation, rather than linearly. A node with 4x the reputation of another gets only 2x the voting power.

11 Conclusion

AGI Expert represents a fundamentally different approach to AI research infrastructure. Rather than centralizing compute, data, and governance in a single institution, the network distributes all three across a permissionless mesh of autonomous agents.

The design provides the following guarantees:

  1. Trustless token economics. The AGIEX token has no pre-mine, no team allocation, and no admin keys. Every token is minted on-chain in response to a verified proof. The mint authority is a PDA, and the program is deployed immutably.
  2. Verifiable computation. The pulse protocol combines matrix computation challenges, Merkle commitment schemes, VRF leader election, and multi-party cross-validation to ensure that only genuine compute earns rewards.
  3. Layered security. Ten on-chain security layers, exponential cooldowns, slashing, and permanent bans make attacks economically irrational. P2P security adds rate limiting, challenge-response liveness, anomaly detection, and automatic blocklisting.
  4. Adaptive participation. Hardware auto-detection and the nine-capability system allow any device from a browser to an H100 server to participate meaningfully and earn rewards proportional to its contribution.
  5. Autonomous research. Eight parallel research projects run continuously across every node, with evolutionary mutation strategies, CRDT-based result sharing, and global leaderboards that converge without a central server.
  6. Diversified governance. A seven-seat council spanning AI/ML, engineering, ethics, law, science, community advocacy, and security ensures that research direction reflects broad expertise rather than narrow interests.
